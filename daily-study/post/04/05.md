### ✅ 3.5 관계 연산자 (Relational Operators)

- ==, !=, >, <, >=, <= — 결과는 항상 `bool`
- = (대입) vs == (비교) 혼동은 고전적 버그 원인

> [!warning] 부동소수점 비교
> ```cpp
> // ❌ 절대 하면 안 됨
> if (0.1 + 0.2 == 0.3)
>
> // ✅ epsilon으로 비교
> if (abs(a - b) < 1e-6)
> ```
> 실수(float/double)는 저장 방식상 오차가 생김 → 직접 == 금지

### ✅ 3.6 논리 연산자 (Logical Operators)

| 연산자    | 의미          | 예시                   |
| ------ | ----------- | -------------------- |
| `&&`   | AND (둘 다 참) | `hit && hp < 20`     |
| `\|\|` | OR (하나라도 참) | `v == 0 \|\| v == 1` |
| `!`    | NOT (반전)    | `!isDead`            |

- **우선순위**: `&&` > `||` → 헷갈리면 괄호 필수!
- **XOR 구현**: `(a != b)` — C++에는 논리 XOR 없음
- **De Morgan**: `!(A && B)` = `!A || !B`

> [!caution] Short-Circuit Evaluation ⭐ 중요
> ```cpp
> int y = 2;
> if (x == 1 && y++ == 2) { }  // x가 false면 y++는 실행 안 됨!
> // x가 2면 y는 여전히 2 (예상과 다름)
> ```
> `&&` 왼쪽이 false면 오른쪽은 **아예 평가 안 함**

### ✅ 3.7 이진수 (Binary Numbers)

```
2진수 각 자리 = 2의 거듭제곱
0101 1110 = 0+64+0+16+8+4+2+0 = 94

음수 (2의 보수):
  5  = 0000 0101
NOT  = 1111 1010
+1   = 1111 1011 = -5 ✓
```

- **10진수 → 2진수**: 2로 계속 나누며 나머지 역순으로 읽기
- **signed vs unsigned**: 맨 앞 비트가 부호로 쓰이면 같은 이진수도 결과가 다름!

```
1001 1110
  signed → -98
unsigned → 158  (같은 비트, 완전히 다른 값!)
```

### ✅ 3.8 비트단위 연산자 (Bitwise Operators)

| 연산자 | 이름 | 의미 |
|--------|------|------|
| `<<` | Left Shift | 비트 왼쪽으로 이동 = **× 2ⁿ** |
| `>>` | Right Shift | 비트 오른쪽으로 이동 = **÷ 2ⁿ** |
| `~` | NOT | 비트 전체 반전 (≠ 논리 NOT `!`) |
| `&` | AND | 둘 다 1이면 1 |
| `\|` | OR | 하나라도 1이면 1 |
| `^` | XOR | 다르면 1, 같으면 0 |

```cpp
unsigned int a = 0b1100;  // 12
unsigned int b = 0b0110;  // 6

cout << bitset<4>(a & b); // 0100 (4)
cout << bitset<4>(a | b); // 1110 (14)
cout << bitset<4>(a ^ b); // 1010 (10)

// 시프트 = 2의 거듭제곱 연산 (곱하기/나누기보다 빠름)
3 << 3 == 24  // 3 × 2³
```

> [!warning] 논리 vs 비트 연산자 헷갈리지 말 것!
> - `!x` = 논리 NOT (bool 결과)
> - `~x` = 비트 NOT (각 비트 반전)
> - `&&` = 논리 AND / `&` = 비트 AND

### ✅ 3.9 비트플래그 & 비트마스크

**비트 플래그**: 여러 bool을 하나의 정수에 저장

```cpp
// 8개 아이템 상태를 1바이트로 표현
const unsigned char ITEM_0 = 1 << 0;  // 0b00000001
const unsigned char ITEM_1 = 1 << 1;  // 0b00000010
const unsigned char ITEM_2 = 1 << 2;  // 0b00000100

unsigned char inventory = 0;

inventory |= ITEM_0;           // 아이템 0 획득 (설정)
if (inventory & ITEM_0) { }    // 아이템 0 보유 확인
inventory &= ~ITEM_0;          // 아이템 0 제거
inventory ^= ITEM_2;           // 아이템 2 토글
```

**비트 마스크**: 특정 비트 범위 추출

```cpp
// RGB 색상 추출
unsigned int pixel = 0xDAA520;
const unsigned int RED_MASK   = 0xFF0000;
const unsigned int GREEN_MASK = 0x00FF00;
const unsigned int BLUE_MASK  = 0x0000FF;

unsigned char r = (pixel & RED_MASK)   >> 16;  // 218
unsigned char g = (pixel & GREEN_MASK) >> 8;   // 165
unsigned char b = (pixel & BLUE_MASK)  >> 0;   // 32
```

**RGBA (알파 채널 추가)**:

```cpp
// 0xAARRGGBB 구조
const unsigned int ALPHA_MASK = 0xFF000000;  // 비트 24-31

// 채널 합치기 (패킹)
unsigned int color = ((unsigned int)a << 24)
                   | ((unsigned int)r << 16)
                   | ((unsigned int)g << 8)
                   | (unsigned int)b;
```

## 2️⃣ Key Insights

- **이진수를 알아야 비트 연산이 보인다** — 3.7이 3.8~3.9의 토대
- **short-circuit evaluation**은 논리 연산자의 핵심 — 사이드 이펙트 주의
- **비트 플래그**는 게임 개발의 필수 기법 (bool 8개 → 1바이트, 함수 파라미터 1개)
- **16진수는 이진수의 줄임 표현** — 1자리 hex = 4비트 nibble, RGB에 최적
