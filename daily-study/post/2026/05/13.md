## 1️⃣ SGD Regressor 복습 — 선형 회귀의 점진적 학습

**SGD (Stochastic Gradient Descent)**: 전체 데이터 대신 **1개 샘플씩 가중치를 업데이트**하는 경사하강법.

```python
from sklearn.linear_model import SGDRegressor

sgd_regressor = SGDRegressor(max_iter=1, tol=None, learning_rate='constant')

for iteration in range(5001):
    sgd_regressor.partial_fit(X, Y)   # 점진적 학습 (온라인 러닝)
    predictions  = sgd_regressor.predict(X)
    loss         = np.mean((predictions - Y) ** 2)   # MSE
```

**5000번 반복 수렴 결과:**

| 반복 | 가중치(Weight) | 편향(Bias) | 손실(Loss) |
|------|-------------|----------|----------|
| 0 | ~2270 | ~185 | ~4,458,281 |
| 500 | ~1998 | ~24.8 | ~72 |
| 2500 | ~1999.99 | ~0.088 | ~0.0028 |
| 5000 | ~1999.99 | ~0.009 | ~0.000026 |

- **최종 예측**: 사과 7개 → **14,000.04원** (실제 단가 2,000원/개에 수렴)
- **`partial_fit()`**: 온라인 학습(Online Learning) 지원 — 새 데이터 도착 시 전체 재학습 불필요

| 방식 | 업데이트 단위 | 장점 | 단점 |
|------|------------|------|------|
| Batch GD | 전체 | 안정적 | 대용량에서 느림 |
| SGD | 1개 샘플 | 빠름, 온라인 학습 | 불안정한 수렴 |
| Mini-Batch GD | n개 (32~256) | 속도·안정 균형 | 배치 크기 튜닝 필요 |

## 2️⃣ 로지스틱 회귀 — 분류를 위한 시그모이드

선형 회귀의 출력 범위는 $(-\infty, +\infty)$이므로 0~1 확률이 필요한 **분류에 직접 사용 불가**. 해결책이 **시그모이드 함수**.

$$\sigma(z) = \frac{1}{1 + e^{-z}}, \quad z = w_1 x_1 + w_2 x_2 + \cdots + b$$

| z 값 | σ(z) | 예측 클래스 |
|------|------|----------|
| 매우 작음 | ≈ 0 | 클래스 0 |
| 0 | 0.5 | **결정 경계** |
| 매우 큼 | ≈ 1 | 클래스 1 |

**손실 함수 — Binary Cross-Entropy (Log Loss):**

$$\mathcal{L} = -\frac{1}{n} \sum \left[ y_i \log(\hat{p}_i) + (1 - y_i) \log(1 - \hat{p}_i) \right]$$

> MSE 대신 Log Loss를 쓰는 이유: Log Loss는 **볼록 함수(Convex)** → 전역 최솟값으로 수렴 보장. MSE는 분류에서 비볼록.

```python
from sklearn.linear_model import LogisticRegression

log_r = LogisticRegression()
log_r.fit(X_train, y_train)
y_pred = log_r.predict(X_test)
# 확률값 조회: log_r.predict_proba(X_test)
```

| 피처 | 계수 | 기여 방향 |
|------|------|---------|
| politics | **-1.736** | 클래스 0 강하게 기여 |
| intelligence | **-1.675** | 클래스 0 강하게 기여 |
| land_command | **+1.540** | 클래스 1 강하게 기여 |
| force | +1.168 | 클래스 1 기여 |
| Intercept | 1.059 | 기본 바이어스 |

## 3️⃣ 피처 스케일링 — StandardScaler vs MinMaxScaler

로지스틱 회귀·SVM·신경망은 **피처 크기(Scale)에 민감** → 스케일링 필수.

$$\text{StandardScaler}: \quad x' = \frac{x - \mu}{\sigma} \quad \text{(평균 0, 표준편차 1)}$$

$$\text{MinMaxScaler}: \quad x' = \frac{x - x_{min}}{x_{max} - x_{min}} \quad \text{(0~1 범위)}$$

| 구분 | StandardScaler | MinMaxScaler |
|------|--------------|-------------|
| 결과 범위 | (-∞, +∞), 평균 0 | [0, 1] |
| 이상치 영향 | 적음 | 큼 (범위 왜곡) |
| 사용 알고리즘 | 로지스틱, SVM, PCA | 신경망, KNN |

> [!warning] 데이터 누수(Data Leakage) 주의
> 스케일러는 **Train 데이터로만 `fit()`**, Test 데이터는 Train의 통계값으로 `transform()`만 수행. Test로 fit하면 미래 정보가 Train에 유입 → 실제보다 높은 성능 측정.

## 4️⃣ 분류 평가 지표 — 혼동행렬부터 MCC까지

### 혼동행렬 (Confusion Matrix)

```
                예측 P    예측 N
실제 P    │   TP    │   FN   │
실제 N    │   FP    │   TN   │
```

**week10 결과**: TP 43, TN 29, FP 0, FN 3

### 핵심 지표 수식

$$\text{Precision} = \frac{TP}{TP+FP} \quad \text{(예측 양성 중 실제 양성)}$$

$$\text{Recall} = \frac{TP}{TP+FN} \quad \text{(실제 양성 중 탐지한 비율)}$$

$$\text{F1} = \frac{2 \times \text{Precision} \times \text{Recall}}{\text{Precision}+\text{Recall}} \quad \text{(조화 평균)}$$

$$\text{MCC} = \frac{TP \cdot TN - FP \cdot FN}{\sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)}}$$

**week10 로지스틱 회귀 결과:**

| 지표 | 클래스 0 | 클래스 1 | 전체 |
|------|---------|---------|------|
| Precision | 0.91 | **1.00** | — |
| Recall | **1.00** | 0.93 | — |
| F1 | 0.95 | 0.97 | — |
| Accuracy | — | — | **0.96** |
| MCC | — | — | **0.920** |

**어떤 지표를 언제 쓰는가:**

| 상황 | 우선 지표 | 이유 |
|------|---------|------|
| 암 진단, 사기 탐지 | **Recall** | FN(놓침) 최소화 |
| 스팸 필터 | **Precision** | FP(정상→스팸) 최소화 |
| 클래스 불균형 | **MCC** | 4가지 오류 모두 반영 |
| 일반 균형 데이터 | F1 또는 Accuracy | — |

### 차원 축소 3종 비교

13개 피처를 **2D로 압축**하여 클러스터 구조 시각화.

```python
from sklearn.decomposition import PCA
from sklearn.manifold import TSNE
import umap.umap_ as umap

X_pca  = PCA(n_components=2).fit_transform(X_fit_Z)
X_tsne = TSNE(n_components=2).fit_transform(X_fit_Z)
X_umap = umap.UMAP(n_components=2).fit_transform(X_fit_Z)
```

| 알고리즘 | 보존 구조 | 속도 | 재현성 | 특징 |
|---------|---------|------|--------|------|
| **PCA** | 전역 분산 (선형) | 빠름 | 결정론적 | 해석 가능, 주성분 |
| **t-SNE** | 국소 이웃 관계 | 느림 O(n²) | 비결정론적 | 클러스터 시각화 최강 |
| **UMAP** | 전역 + 국소 균형 | t-SNE보다 5배↑ | 비결정론적 | 위상 기하학 기반 |

> [!tip] 같은 데이터, 다른 클러스터 모양
> PCA·t-SNE·UMAP을 같은 심장병 데이터에 적용했을 때 클러스터의 형태가 전혀 다르게 나타났다. 각 알고리즘이 "무엇을 보존하는가"의 철학 차이가 시각화로 드러난다. 단일 차원 축소 결과만 믿지 말 것.

## 6️⃣ 트리 기반 앙상블 모델 비교

### 결정트리 (Decision Tree)

```python
tree_clf = DecisionTreeClassifier(max_depth=4)
tree_clf.fit(X_train, y_train)
```

- 분기 기준: **지니 계수(Gini Impurity)** 최소화
  $$\text{Gini} = 1 - \sum p_k^2 \quad (0:\text{완전 순수}, 0.5:\text{최대 불순})$$
- **graphviz**로 트리 전체 구조 시각화 가능 (White-box 모델)
- 결과: Accuracy 0.79, MCC 0.586

### 랜덤 포레스트 (Random Forest) — 배깅 앙상블

```python
rf_clf = RandomForestClassifier(n_estimators=100)
```

- **배깅(Bagging)**: 복원 추출로 100개의 서로 다른 트리 학습
- 각 분기에서 **랜덤 피처 서브셋만** 고려 → 트리 간 상관관계 감소
- OOB(Out-Of-Bag): 부트스트랩에 미포함된 ~37% 데이터로 자동 검증
- 결과: Accuracy **0.85**, MCC **0.695** ← **전 모델 중 최고**

### XGBoost — 2차 근사 부스팅

```python
xgb_clf = xgb.XGBClassifier(n_estimators=100)
```

- **부스팅(Boosting)**: 이전 트리의 오류(잔차)를 다음 트리가 집중 수정
- 손실 함수 2차 Taylor 전개 → 최적 잎 값을 수식으로 계산 (빠른 수렴)
- L1 + L2 정규화로 과적합 방지
- 결과: Accuracy 0.84, MCC 0.668

### LightGBM — Leaf-wise 고속 부스팅

```python
lgb_clf = lgb.LGBMClassifier(n_estimators=100)
```

- **Leaf-wise 성장**: 손실 감소가 가장 큰 리프를 우선 분기 (XGBoost Level-wise 대비 빠름)
- **GOSS**: 기울기 작은 샘플 일부만 사용 → 속도↑, 정확도 유지
- 결과: Accuracy 0.84, MCC 0.668

### 모델별 성능 비교

| 모델 | Accuracy | MCC | Feature Importance 1위 |
|------|---------|-----|----------------------|
| Logistic Regression | 0.82 | 0.644 | politics(-1.736) |
| Decision Tree | 0.79 | 0.586 | **cp (0.40)** |
| **Random Forest** | **0.85** | **0.695** | cp (상위권) |
| XGBoost | 0.84 | 0.668 | **cp (0.22)** |
| LightGBM | 0.84 | 0.668 | **age** (다름!) |

> [!info] LightGBM Feature Importance가 다른 이유
> LightGBM 기본 `importance_type`은 `'split'`(분기 횟수 기준), XGBoost는 `'weight'`. 같은 데이터라도 중요도 정의가 달라 순위가 달라진다. `importance_type='gain'`으로 통일하면 비교 가능.

## 7️⃣ 셰이더 디자이너 · 그래픽스 엔지니어 관점

> [!example] 관점 1: DLSS·FSR은 이 수업에서 배운 모델들의 실전 적용이다
> Nvidia DLSS는 저해상도 프레임 + 모션 벡터를 입력으로 받아 고해상도를 출력하는 **CNN(합성곱 신경망)** 기반 업스케일러다. AMD FSR 4도 머신러닝 기반이다. 오늘 배운 로지스틱 회귀·트리 모델의 원리 — 입력 피처에서 출력 클래스(또는 값)를 예측 — 가 렌더링 파이프라인 안에 이미 들어와 있다. 셰이더 디자이너가 "AI 렌더링"을 이해하려면 이 수업의 기초가 필수다.

> [!example] 관점 2: Feature Importance는 G-Buffer 채널 설계에 대한 힌트를 준다
> Deferred Rendering의 G-Buffer에는 Albedo·Normal·Roughness·Metallic 등 여러 채널이 있다. 어떤 채널이 최종 조명 품질에 가장 큰 영향을 주는지를 분석하는 것은 **Feature Importance 분석과 동일한 발상**이다. 실제로 연구자들은 렌더링 품질 저하 없이 G-Buffer 채널을 줄이기 위해 머신러닝 기반 중요도 분석을 사용한다. 오늘의 `plot_feature_importances()`가 그 개념의 기초다.

> [!example] 관점 3: PCA는 텍스처 압축의 수학적 조상이다
> BC7·ASTC 같은 GPU 텍스처 압축 포맷은 **블록 단위로 색상 팔레트를 선형 부분공간으로 근사**한다. 이것은 PCA가 고차원 데이터를 분산 최대 보존 방향으로 투영하는 원리와 같다. 오늘 PCA로 13차원 심장병 데이터를 2차원으로 축소한 것처럼, 텍스처 압축기는 64개 픽셀의 색상을 2개의 기저 벡터(엔드포인트)로 근사한다.

> [!example] 관점 4: 앙상블 렌더링 — 여러 약한 판단의 합산
> 랜덤 포레스트의 "100개 트리의 다수결"과 TAA(Temporal Anti-Aliasing)의 "여러 프레임 샘플의 누적 평균"은 같은 철학이다. 단일 판단(단일 프레임 샘플)은 노이즈가 많지만, 여러 판단의 평균(앙상블·시간 누적)은 노이즈가 줄고 안정된다. 레이 트레이싱의 경로 추적(Path Tracing)에서 수천 개의 광선 샘플을 누적하는 Monte Carlo Integration도 동일한 원리다.

> [!success] 인사이트 2: 앙상블은 편향-분산 트레이드오프를 하드웨어로 해결한다

> [!success] 인사이트 3: 차원 축소는 "무엇을 버릴지"의 철학이다
> PCA는 분산을, t-SNE는 이웃 관계를, UMAP은 위상 구조를 보존하면서 나머지를 버린다. 같은 데이터가 세 가지 완전히 다른 모습으로 시각화되는 것을 보며, 데이터 분석은 객관적인 진실 탐구가 아니라 **"어떤 관점으로 볼 것인가"의 선택**임을 느꼈다. 셰이더를 설계할 때도 어떤 정보를 G-Buffer에 담고 무엇을 버릴지 결정하는 것과 같은 문제다.

> [!success] 인사이트 4: cp (흉통 유형)의 압도적 1위는 도메인 지식의 검증이다
> Decision Tree·Random Forest·XGBoost 모두 `cp (chest pain type)`을 Feature Importance 1위로 선택했다. 실제 심장내과에서도 흉통은 심장병 진단의 첫 번째 지표다. 머신러닝이 **데이터에서 스스로 의학적 도메인 지식과 일치하는 결론에 도달**했다는 것이 인상 깊었다. 좋은 모델은 단순히 숫자가 높은 게 아니라, 실제 현실과 일치하는 패턴을 발견하는 것이다.
