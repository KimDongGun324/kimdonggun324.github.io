## 1️⃣ Part A — C++ 연산자 (3.1 ~ 3.4)

### 3.1 연산자 우선순위와 결합 법칙 (Operator Precedence & Associativity)

- 우선순위 16단계 테이블: 괄호`()` → 쉼표`,` 순서
- **좌결합(Left-to-Right)** vs **우결합(Right-to-Left)** 명확히 구분
- 복합 표현식에서 계산 순서를 예측하고 제어 가능

### 3.2 산술 연산자 (Arithmetic Operators)

- **정수 나눗셈** vs **실수 나눗셈** 차이 주의

```cpp
7 / 4     // = 1   (정수: 소수점 버림)
7 / 4.0f  // = 1.75 (실수: float 연산)
```

- 나머지 연산자 `%` — 부호는 **피제수(dividend)** 의 부호를 따름
- 복합 대입 연산자 (`+=`, `-=`, `*=`, `/=`) → 코드 간결화

### 3.3 증감 연산자 (Increment / Decrement Operators)

| 구분 | 동작 | 반환값 |
|------|------|--------|
| `++x` (전위, Prefix) | 먼저 증가 후 반환 | 증가된 값 |
| `x++` (후위, Postfix) | 반환 후 증가 | **이전 값** (임시 복사본 생성) |

- **성능**: 후위는 임시 복사본 생성 → **전위 권장**
- ⚠️ **Undefined Behavior**: `x = x++` 같은 표현 절대 금지

### 3.4 `sizeof`, 쉼표, 조건부 연산자

- `sizeof`: **컴파일 타임**에 메모리 크기 반환 (런타임 비용 없음)
- 쉼표 연산자: 여러 표현식 순차 평가 → **마지막 값만** 반환
- 조건부(삼항, Ternary) 연산자:

```cpp
// if-else보다 간결, const 초기화에도 사용 가능
int result = (a > b) ? a : b;
```

## 2️⃣ Part B — 벡터 수학 (Vector Mathematics)

### 벡터 핵심 개념

- **벡터(Vector)** = 방향(Direction) + 크기(Magnitude)
- 점(Point)과 구분: 벡터는 이동량, 점은 절대 위치

### 벡터 연산 요약

| 연산 | 수식 | 결과 타입 | 용도 |
|------|------|-----------|------|
| 덧셈/뺄셈 | $\vec{A} \pm \vec{B}$ | 벡터 | 위치 이동 |
| 스칼라 곱 | $k\vec{A}$ | 벡터 | 크기 조절 |
| 내적 (Dot Product) | $\vec{A} \cdot \vec{B} = \|A\|\|B\|\cos\theta$ | **스칼라** | 각도, 투영 |
| 외적 (Cross Product) | $\vec{A} \times \vec{B}$ | **벡터** (3D 전용) | 법선 벡터 |

- **핵심 성질**: $\vec{A} \cdot \vec{B} = 0 \Longleftrightarrow \vec{A} \perp \vec{B}$ (수직)

### GLM (OpenGL Mathematics) 주요 함수

```cpp
glm::length(v)       // 크기 |v| = sqrt(v·v)
glm::normalize(v)    // 단위 벡터 (크기 = 1)
glm::dot(a, b)       // 내적
glm::cross(a, b)     // 외적 (3D 전용)
```

#### ⚡ 성능 최적화 — `sqrt()` 제거

```cpp
// ❌ 느림: sqrt() 연산 발생
float dist = glm::length(a - b);

// ✅ 빠름: 거리² 비교로 sqrt() 생략
float distSq = glm::dot(a - b, a - b);
if (distSq < radiusSq) { /* ... */ }
```

### 7강 🌟 — 좌표 변환 (Coordinate Transform) — 핵심!

#### 두 좌표계 비교

| 항목 | 스크린 좌표계 (Screen Space) | 월드 좌표계 (World Space) |
|------|------------------------------|--------------------------|
| X 범위 | $0 \sim \text{width}-1$ | $-\text{aspect} \sim +\text{aspect}$ |
| Y 범위 | $0 \sim \text{height}-1$ | $-1 \sim +1$ |
| 원점 위치 | 좌상단 | 중앙 |
| Y 증가 방향 | 아래 ↓ | 위 ↑ |

#### 변환 공식 (Screen → World)

```cpp
const float aspect = float(width) / height;
const float xScale = 2.0f * aspect / (width - 1);
const float yScale = 2.0f / (height - 1);

glm::vec2 screenToWorld(glm::vec2 posScreen) {
    return glm::vec2(
        posScreen.x * xScale - aspect,           // X: 스케일 → 이동
        -(posScreen.y * yScale - 1.0f)           // Y: 스케일 → 이동 → 반전
    );
}
```

#### 3단계 변환 흐름

```
[1] Scale   → 범위 조정 (픽셀 단위 → 정규화)
[2] Offset  → 원점 이동 (좌상단 → 중앙)
[3] Flip    → Y축 반전 (↓ → ↑)
```

## 💡 Key Insights

### C++ 연산자 설계의 정교함
- 16단계 우선순위는 단순 암기가 아닌 **표현식 설계 능력**의 기반
- Undefined Behavior는 컴파일 오류보다 **더 위험** — 디버깅이 불가능
- 전위/후위 차이는 작지만, 루프 최적화에서 원칙의 문제

### 벡터는 그래픽스의 언어
- 덧셈 · 내적 · 외적 **3가지**만으로 거의 모든 렌더링 계산 가능
- $\vec{A} \cdot \vec{B} = 0$ → 수직 판별은 조명·셰이딩에서 매일 쓰임
- GLM 없이도 원시 계산 가능해야 **진짜 이해** (HLSL에서 직접 구현)

### 좌표 변환은 그래픽스의 뼈대
- 2D 변환을 완벽히 이해하면 **3D MVP 행렬**도 원리는 동일
- Aspect Ratio = 단순 화면 비율이 아닌 **공간 왜곡 방지**의 핵심
- Vertex Shader의 Clip Space 변환과 직결되는 개념
