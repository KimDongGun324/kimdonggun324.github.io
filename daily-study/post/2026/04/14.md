### ✅ [C++] void 무치형 (2.4)

`void`는 **"값이 없음"** 을 나타내는 특수 타입. 크기가 0이라 변수 선언은 불가하지만, 타입 시스템에서 3가지 역할을 한다.

```cpp
void my_function(void) {}   // ① 파라미터 없음 명시 (C 스타일, C++에선 생략 가능)
void print() {}             // ② 반환값 없는 함수

// void my_var;  ❌ — 크기 없는 타입은 변수 선언 불가

void* my_void;              // ③ void 포인터 — 어떤 타입의 주소든 저장 가능
int   i = 123;
float f = 123.456f;
my_void = (void*)&i;        // C 스타일 형변환으로 저장
my_void = (void*)&f;
```

> **포인터 이해의 핵심**: 데이터 타입의 크기가 달라도 그 **주소를 표현하는 크기는 동일**하다.
> 64비트 시스템에서 `sizeof(int*)` == `sizeof(double*)` == `sizeof(void*)` == **8 bytes**

| 용도 | 문법 | 가능 여부 |
|---|---|---|
| void 변수 선언 | `void v;` | ❌ |
| void 반환 함수 | `void f() {}` | ✓ |
| void 포인터 선언 | `void* p;` | ✓ |
| void* 역참조 | `*p` | ❌ (캐스팅 후 가능) |

### ✅ [C++] 열거형 enumerated types (4.7)

연관된 상수들을 하나의 그룹으로 묶어 이름을 붙이는 방법.

```cpp
enum Color { RED, ORANGE, YELLOW };   // RED=0, ORANGE=1, YELLOW=2 (자동 할당)

Color my_color = RED;
cout << my_color;   // 0 (정수로 출력)

// 값 직접 지정도 가능
enum Color { RED = -3, ORANGE, YELLOW = 10 };  // ORANGE = -2
```

**주의사항:**
- 서로 다른 enum 간 직접 비교 가능 → 의도치 않은 비교 버그 유발
- 같은 스코프 내 이름 충돌 발생 가능 (`RED`가 두 enum에 있으면 에러)
- `int`로의 암시적 변환은 가능하지만, `int → enum`은 `static_cast` 필요

```cpp
Color c = static_cast<Color>(2);    // int → enum 명시적 캐스팅 필요
```

### ✅ [C++] 영역 제한 열거형 enum class (4.8)

C++11에서 도입된 **scoped enum**. 기존 `enum`의 단점(이름 충돌, 타입 안전성 부재)을 해결.

```cpp
enum class Color { RED, BLUE, GREEN };
enum class Fruit { APPLE, BANANA };

Color c = Color::RED;        // ✅ 반드시 범위 지정자(::) 필요
// Color c = RED;            // ❌ 컴파일 에러

// 다른 enum class 간 비교 불가 — 타입 안전
// if (c == Fruit::APPLE) {} // ❌ 컴파일 에러
```

| 항목 | `enum` | `enum class` |
|---|---|---|
| 이름 충돌 | 발생 가능 | ✅ 방지 (스코프 분리) |
| 타입 안전성 | ❌ int 혼용 가능 | ✅ 엄격한 타입 구분 |
| int 암시적 변환 | ✅ 가능 | ❌ static_cast 필요 |
| 사용 방식 | `RED` | `Color::RED` |

> [!tip] 신규 코드에서는 항상 `enum class` 사용을 권장

### ✅ [C++] 타입 별칭 typedef / using (4.9)

복잡한 타입에 **의미 있는 이름(별칭)** 을 붙이는 기능.

```cpp
// C 스타일 — typedef
typedef double distance_t;
typedef vector<pair<string, int>> pairlist_t;

// C++11 모던 스타일 — using (권장)
using distance_t = double;
using pairlist_t = vector<pair<string, int>>;

distance_t d = 1.5;    // double d = 1.5; 와 동일
```

**`_t` 네이밍 컨벤션**: 타입 별칭에 `_t` 접미사를 붙여 "이건 타입 별칭이다"를 표시

**고정 폭 정수 (Fixed-width integers)**: `<cstdint>` 헤더에서 제공하는 플랫폼 독립적 정수 타입

```cpp
#include <cstdint>
std::int8_t  i8;    // 정확히 8bit  (내부적으로 typedef 구현)
std::int32_t i32;   // 정확히 32bit
std::int64_t i64;   // 정확히 64bit
```

**`typedef` vs `using` 결정적 차이 — 템플릿 별칭:**

```cpp
// typedef는 템플릿 별칭 불가 ❌
// template<typename T> typedef vector<T> Vec;  // 컴파일 에러!

// using은 가능 ✅
template<typename T>
using Vec = vector<T>;

Vec<int> v1;      // vector<int>
Vec<string> v2;   // vector<string>
```

### ✅ [C++] 구조체 struct (4.10)

서로 다른 타입의 변수들을 하나로 묶는 **사용자 정의 자료형(user-defined type)**.

```cpp
struct Person
{
    double height = 0.0;  // 인클래스 기본값 (C++11)
    float  weight = 0.0f;
    int    age    = 0;
    string name   = "Unknown";

    void print()    // 멤버 함수 — 데이터와 기능을 함께 묶음
    {
        cout << height << " " << weight << " " << age << " " << name << endl;
    }
};

Person me{ 2.0, 100.0f, 20, "KIM KIM" };  // 균일 초기화 (C++11)
me.print();        // 멤버 선택 연산자 . 으로 멤버 함수 호출
cout << me.age;    // 멤버 변수 접근
```

**멤버 선택 연산자 `.`**: 구조체(또는 클래스)의 멤버에 접근하는 연산자. 클래스에서도 동일하게 사용.

**초기화 방법 비교:**

| 방법 | 코드 | 시대 |
|---|---|---|
| 멤버 개별 대입 | `me.age = 20; me.name = "Kim";` | 구식 |
| 균일 초기화 | `Person me{ 2.0, 100.0f, 20, "Kim" };` | C++11 ✅ |
| 인클래스 기본값 | `int age = 0;` (구조체 선언 시) | C++11 ✅ |

**구조체를 함수에서 활용:**

```cpp
void printPerson(const Person& ps) { ... }  // 파라미터로 전달
Person getMe() { return { 2.0, 100.0f, 20, "Kim" }; }  // 반환값으로 사용

// 중첩 구조체
struct Family { Person me, mom, dad; };
Family f;
f.me.age = 20;  // 점 연산자 체이닝
```

**대입 연산자 주의**: 포인터 멤버가 있는 복잡한 구조체는 `me2 = me` 단순 대입 시 **얕은 복사(shallow copy)** 문제 발생 → 복사 생성자/대입 연산자 오버로딩 필요 (클래스 챕터에서 상세 학습)

**`sizeof`와 패딩(Padding):**

```cpp
struct Employee { short id; int age; double wage; };
// 예상: 2+4+8 = 14 bytes
cout << sizeof(Employee);  // 실제: 16 bytes!
```

```
메모리 배치:
[short 2B][패딩 2B][int 4B][double 8B] = 16 bytes
```

CPU가 데이터를 효율적으로 읽으려면 **특정 배수 주소에 데이터를 배치(정렬, Alignment)** 해야 하기 때문에 컴파일러가 빈칸(패딩)을 자동 삽입한다. `sizeof` 값이 멤버 합산과 다를 수 있다는 것을 기억할 것.

> [!note] 구조체 → 클래스로 가는 길
> `struct`에서 멤버 함수를 배우면 C++ 클래스(OOP) 개념이 자연스럽게 이어진다.
> 기본 접근 지정자만 다를 뿐(`struct`=public 기본, `class`=private 기본) 나머지는 동일.

### ✅ [셰이더] 셰이더의 정의와 개요 (1강)

**셰이더(Shader)**: 렌더링 파이프라인에서 GPU가 실행하는 **특수 목적 프로그램**. 정점(Vertex)의 위치나 픽셀(Pixel)의 색깔을 계산하는 함수.

**최소 필수 셰이더 2종:**

| 셰이더 | 역할 | 처리 단위 |
|---|---|---|
| **버텍스 셰이더 (Vertex Shader)** | 3D 정점 좌표를 2D 화면 좌표로 변환 | 정점(Vertex)마다 |
| **픽셀 셰이더 (Pixel Shader)** | 각 픽셀의 최종 색상 결정 (조명, 텍스처, 반사 등) | 픽셀마다 |

**셰이더가 빠른 이유 — GPU의 대규모 병렬 처리:**

```
CPU: 복잡한 작업을 순서대로 처리 (직렬)
GPU: 수백만 픽셀에 대해 셰이더를 동시에 실행 (대규모 병렬)
     → 셰이더는 "낱개의 픽셀마다 실행" + "이 실행이 모두 한 번에 일어남"
```

> [!info] 핵심 관점 전환
> 셰이더 코드를 작성한다는 것은 "하나의 픽셀이 어떻게 처리될지"를 정의하는 것.
> GPU가 그 코드를 수백만 픽셀에 동시에 적용한다.

## 2️⃣ Key Insights

- **`enum class`는 그냥 `enum`의 업그레이드** — 이름 충돌 방지 + 타입 안전성, 신규 코드에서는 항상 `enum class`
- **`typedef` vs `using` — 결정적 차이는 템플릿** — 단순 별칭은 동일하지만 `template<typename T> using Vec = ...`은 `typedef`로 불가능
- **`_t` 접미사는 "이건 타입이에요" 라는 신호** — `std::int32_t` 같은 표준 타입도 내부적으로 `typedef`로 구현
- **구조체 멤버 함수 = OOP의 시작** — `me.print()` 처럼 데이터와 기능을 묶는 순간 클래스/객체 지향의 본질에 발을 들인 것
- **`sizeof(struct)` ≠ 멤버 크기 합** — 패딩 때문에 항상 더 클 수 있다. 네트워크 패킷/파일 포맷 처리 시 특히 주의
- **포인터의 핵심: 주소 크기는 타입과 무관** — 이것 하나 이해하면 `void*`가 왜 존재하는지, 왜 모든 포인터가 같은 크기인지 자연히 납득됨

---
- **셰이더 = "1픽셀에 대한 함수" × 수백만 번** — 병렬성이 셰이더 성능의 전부
