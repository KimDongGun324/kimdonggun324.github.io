### ✅ [C++] 난수 만들기 (5.9)

#### 난수의 3가지 분류

| 종류 | 설명 | 예시 |
|------|------|------|
| **TRNG** (True Random) | 물리적 엔트로피 — 예측 불가 | 방사선 붕괴, `std::random_device` |
| **PRNG** (Pseudo Random) | 수식으로 생성 — 시드가 같으면 동일 수열 | `rand()`, `mt19937` |
| **CSPRNG** (Cryptographically Secure) | 암호학적 안전성 보장 | `BCryptGenRandom`, `/dev/urandom` |

> [!important] 게임·그래픽스 실무에서는 PRNG로 충분하다. CSPRNG는 느리고, 셰이더에서는 아예 사용 불가.

---

#### 직접 구현 — 선형 합동 생성기 (LCG)

```cpp
unsigned int PRNG()
{
    static unsigned int seed = 5523;  // static: 호출 간 상태 유지 (Data Segment)
    seed = 8253729 * seed + 2396403;  // LCG 점화식: X_(n+1) = a*X_n + c
    return seed % 32768;              // [0, 32767] 범위 클리핑
}
```

**핵심 두 가지:**
- `static` 지역 변수 → 함수가 반환돼도 값이 유지됨 (Data Segment, 프로그램 종료까지 존속)
- `unsigned int` 오버플로우 → **암묵적 mod 2³²** 역할 — 이것이 LCG의 `mod m`을 대체

**LCG 수식:**

$$X_{n+1} = (a \cdot X_n + c) \bmod m$$

#### C 스타일: srand / rand

```cpp
// 고정 시드 — 디버깅용 (매 실행 동일 수열)
std::srand(5323);

// 시간 기반 시드 — 릴리즈용 (매 실행 다른 수열)
std::srand(static_cast<unsigned int>(std::time(0)));
// static_cast 이유: time_t (보통 long long) → unsigned int 명시적 변환

for (int count = 1; count <= 100; ++count)
    std::cout << std::rand() << "\t";
```

| 항목 | 설명 |
|------|------|
| `RAND_MAX` | MSVC: 32767 / GCC: 2³¹-1 (플랫폼 의존) |
| 1초 이내 연속 실행 | 같은 시드 → 같은 수열 (time의 정밀도 한계) |
| Y2K38 문제 | 2038년 이후 time_t 오버플로우 가능 |

#### getRandomNumber — fraction 기법 (Modulo Bias 해결)

```cpp
int getRandomNumber(int min, int max)
{
    static const double fraction = 1.0 / (RAND_MAX + 1.0); // 한 번만 계산해 캐시
    return min + static_cast<int>((max - min + 1) * (std::rand() * fraction));
}
```

**단계별 변환:**

```
rand()          → [0, RAND_MAX]
× fraction      → [0.0, 1.0)      정규화
× (range)       → [0.0, range)    스케일링
static_cast<int>→ {0, ..., range-1}
+ min           → {min, ..., max} 최종 범위
```

> **Modulo Bias**: `rand() % N`은 `RAND_MAX + 1`이 N으로 나누어 떨어지지 않으면 특정 값이 더 자주 나온다. fraction 기법과 `uniform_int_distribution`이 이를 수학적으로 제거한다.

#### C++11 권장 방식 — 3계층 아키텍처

```cpp
#include <random>

std::random_device rd;                       // 하드웨어 엔트로피 (시드 공급 전용, 느림)
std::mt19937_64 mersenne(rd());              // 64비트 Mersenne Twister 엔진
std::uniform_int_distribution<> dice(1, 6); // [1,6] 균등 분포, 바이어스 없음

for (int count = 1; count <= 20; ++count)
    std::cout << dice(mersenne) << std::endl;
```

| 계층 | 역할 | 특징 |
|------|------|------|
| `random_device` | 하드웨어 엔트로피 소스 | Linux: `/dev/urandom`, Windows: `BCryptGenRandom` |
| `mt19937_64` | PRNG 엔진 | 주기 2¹⁹⁹³⁷-1, 상태 624×32비트 |
| `uniform_int_distribution` | 분포 함수 | rejection sampling으로 Modulo Bias 완전 제거 |

> [!warning] Mersenne Twister는 암호학적으로 안전하지 않다
> 624개 출력값만 관찰하면 내부 상태를 완전히 역산할 수 있다. 비밀번호·토큰 생성에는 절대 사용 금지.

### ✅ [C++] std::cin 더 잘 쓰기 — 입력 에러 처리 (5.10)

#### cin 스트림 상태 플래그 (iostate)

`std::cin`은 단순 입력 함수가 아닌 **상태 머신**이다. 4개의 비트 플래그로 상태를 관리한다.

| 플래그 | 값 | 의미 |
|--------|-----|------|
| `goodbit` | 0 | 정상 상태 |
| `eofbit` | 1 | EOF 도달 |
| `failbit` | 2 | 형식 오류 (int에 문자 입력, 오버플로우 등) |
| `badbit` | 4 | 스트림 I/O 오류 (하드웨어·파일 오류) |

failbit가 한 번 설정되면 이후의 모든 `cin >>` 연산이 **자동으로 no-op** 처리된다.

---

#### 핵심 3함수 — fail / clear / ignore

```cpp
// ① fail(): failbit | badbit 설정 시 true 반환
if (std::cin.fail()) { ... }

// ② clear(): 에러 플래그 초기화 (버퍼는 그대로!)
std::cin.clear();

// ③ ignore(n, delim): 버퍼에서 최대 n개 문자 제거 (delim 포함)
std::cin.ignore(32767, '\n');

// 현대 C++ 권장
#include <limits>
std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
```

> [!danger] 반드시 `clear()` → `ignore()` 순서
> `ignore()`는 스트림에서 **문자를 읽는** 연산이다.
> failbit가 설정된 상태에서는 읽기 연산이 모두 차단되므로, `clear()` 없이 `ignore()`를 호출하면 **버퍼가 비워지지 않아 무한 루프** 발생.

---

#### getInt() — while(true) 입력 검증 패턴

```cpp
int getInt()
{
    while (true)
    {
        std::cout << "Enter an integer number: ";
        int x;
        std::cin >> x;

        if (std::cin.fail())
        {
            std::cin.clear();                                                  // ① 플래그 초기화
            std::cin.ignore(32767, '\n');                                     // ② 버퍼 비우기
            std::cout << "Invalid number, please try again." << std::endl;
        }
        else
        {
            std::cin.ignore(32767, '\n');  // ③ 성공 시에도 '\n' 제거 (중요!)
            return x;
        }
    }
}
```

**③이 왜 필요한가:** 정상 입력 후에도 Enter('\n')가 버퍼에 남는다. 제거하지 않으면 다음 `getline()` 호출 시 빈 문자열이 즉시 반환된다.

#### getOperator() — 입력 계약(Contract) 보장 패턴

```cpp
char getOperator()
{
    while (true)
    {
        std::cout << "Enter an operator (+, -): ";
        char op;
        std::cin >> op;
        std::cin.ignore(32767, '\n');

        if (op == '+' || op == '-')
            return op;  // 유효할 때만 return → "반드시 유효한 값 반환" 계약 보장
        else
            std::cout << "Invalid operator. Please try again." << std::endl;
    }
}
```

**교수님의 설계 철학**: *"사용자가 잘 입력할 때까지 리턴 안 하고 계속 입력을 받겠죠"*
→ 함수의 **사후조건(postcondition)** 을 코드 수준에서 강제하는 방식. 호출 측에서 별도 유효성 검사 불필요.

---

#### if-else → switch 리팩토링

```cpp
// if-else 버전
void printResult(int x, char op, int y)
{
    if      (op == '+') std::cout << x + y << std::endl;
    else if (op == '-') std::cout << x - y << std::endl;
    else                std::cout << "Invalid operator" << std::endl;
}

// switch 버전 (연산자 추가 시 case만 추가 — 유지보수 용이)
void printResult(int x, char op, int y)
{
    switch (op)
    {
    case '+': std::cout << x + y << std::endl; break;
    case '-': std::cout << x - y << std::endl; break;
    default:  std::cout << "Invalid operator" << std::endl;
    }
}
```

| 관점 | if-else | switch |
|------|---------|--------|
| 시간복잡도 | O(n) 순차 비교 | O(1) jump table (컴파일러 최적화 시) |
| 유지보수 | 분기 추가 시 여러 곳 수정 | case 하나 추가로 완결 |
| 연산자 확장 | 검증 조건 + 출력 두 곳 수정 | switch case 하나만 추가 |

---

#### 입력 버퍼 동작 원리

```
키보드 → [OS 커널 TTY 버퍼] → [cin의 streambuf] → cin >> 연산자
```

- `cin >>`는 Enter까지 입력된 데이터를 **버퍼에서 하나씩** 읽어간다
- 버퍼는 여러 `cin >>` 호출 사이에 **지속(persist)** 된다
- "1 2"를 한 줄에 입력하면 버퍼에 `['1', ' ', '2', '\n']`이 남아 다음 `cin >>`에 영향을 준다

### ✅ [그래픽스 엔지니어 / 셰이더 디자이너 관점]

#### 셰이더에서의 난수 — CPU rand()를 쓸 수 없는 이유

GPU 셰이더(GLSL/HLSL)는 **수천 개의 스레드가 동시에 실행**된다. `rand()`처럼 전역 상태에 의존하는 PRNG는 레이스 컨디션을 일으키며, 셰이더 언어에는 내장 rand가 없다. 대신 **UV 좌표나 픽셀 위치를 입력으로 하는 해시 함수**로 독립적인 난수를 생성한다.

```glsl
// GLSL — 가장 널리 쓰이는 해시 기반 noise 패턴
float rand(vec2 uv) {
    return fract(sin(dot(uv, vec2(12.9898, 78.233))) * 43758.5453);
}
```

| 기법 | 특징 | 용도 |
|------|------|------|
| hash-based (위) | 빠름, 상태 없음, 화이트 노이즈 | 일반 노이즈, dissolve 이펙트 |
| Halton sequence | 균일 분포 보장 (Low-discrepancy) | TAA(Temporal Anti-Aliasing) Jitter |
| Sobol sequence | 다차원 균일 분포 | Path Tracing, 몬테카를로 적분 |
| PCG (Unreal 5.1+) | 품질 + 속도 균형 | Unreal PCG Framework |

> [!info] TAA와 난수 품질
> Temporal Anti-Aliasing은 프레임마다 서브픽셀 위치를 이동시켜 여러 프레임을 누적한다. 이때 사용하는 Halton 수열의 품질이 곧 TAA 결과물의 품질을 결정한다. `mt19937`의 분포 품질이 셰이더 품질로 직결되는 지점.

#### 오프라인 도구에서의 cin 에러 처리

커스텀 셰이더 컴파일러 CLI, 오프라인 라이트맵 베이크 도구, 텍스처 처리 도구 등을 개발할 때 `cin.fail()` + `clear()` + `ignore()` 패턴은 그대로 적용된다. 또한 `std::ifstream`도 `std::basic_istream`을 상속하므로 **동일한 fail/clear/ignore API로 SPIR-V, HLSL 소스, 씬 설명 파일을 읽을 때 에러를 처리**할 수 있다.

```cpp
// cin과 동일한 에러 처리 API — ifstream으로 셰이더 소스 읽기
std::ifstream shaderFile("pbr.hlsl");
if (shaderFile.fail())
{
    shaderFile.clear();
    std::cerr << "Failed to open shader file." << std::endl;
}
```
