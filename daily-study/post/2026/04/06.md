## 1️⃣ Today I Learned

### ✅ 2.1 CPU의 기본 구조 (컴퓨터 구조론)

CPU는 세 가지 핵심 부품으로 구성된다.

| 구성 요소              | 역할                               |
| ------------------ | -------------------------------- |
| **ALU** (산술논리연산장치) | 덧셈·뺄셈·곱셈·나눗셈 / AND·OR·NOT 연산     |
| **레지스터 세트**        | CPU 내 가장 빠른 기억장치, 특수/일반 목적용으로 구분 |
| **제어 유니트**         | 명령어 해석 → 제어 신호 순차 발생 → 동작 시간 결정  |

> [!tip] 제어 유니트 복잡도 해결
> - **마이크로프로그래밍**: 소프트웨어로 처리 → 느림
> - **RISC**: 명령어 수를 줄여 하드웨어만으로 빠르게 처리 → 현재 주류

### ✅ 버스 (Bus) — CPU 내부 버스 & 시스템 버스

**버스** = 컴퓨터 부품 간 데이터를 실어 나르는 "고속도로"

| 버스 종류 | 범위 | 특징 |
|-----------|------|------|
| **CPU 내부 버스** | CPU 내부 | ALU·레지스터·제어 유니트 전용, 외부 직접 연결 불가 |
| **시스템 버스** | CPU 외부 | RAM·GPU·HDD 등과 연결되는 메인 고속도로 |

**시스템 버스 3가지 차선:**

| 버스 | 방향 | 역할 |
|------|------|------|
| 주소 버스 | 단방향 → | "어디서?" 번지수 전달 |
| 데이터 버스 | 양방향 ↔ | "무엇을?" 실제 데이터 이동 |
| 제어 버스 | 단방향 → | "어떻게?" 읽기/쓰기 명령 |

> [!warning] CPU 내부 버스가 시스템 버스와 직접 연결 안 되는 이유
> 1. **속도 차이** — CPU(GHz) vs 시스템 버스(MHz) 클럭 불일치
> 2. **전기적 신호 차이** — 노이즈 유입, 전압 불일치, 신호 충돌
> 3. **보호** — 내부 연산 중 외부 차단 필요
> 4. **데이터 폭 차이** — 비트 수 변환 필요
> → **버퍼 레지스터 / 인터페이스 회로** 가 반드시 필요

### ✅ CPU 작동 원리 (트랜지스터 → 명령어 처리)

**밑바닥부터 CPU까지의 흐름:**

```
트랜지스터 (ON/OFF 스위치)
    ↓
논리 게이트 (AND / OR / NOT / XOR)
    ↓
덧셈 회로 → ALU
    ↓
레지스터 + 제어 유니트
    ↓
명령어 처리 4단계
```

**CPU 명령어 처리 4단계:**

| 단계 | 이름 | 내용 |
|------|------|------|
| ① | **인출 (Fetch)** | PC → MAR → MBR → IR, 명령어를 메모리에서 가져옴 |
| ② | **해석 (Decode)** | IR → 제어 유니트, 명령어 의미 해석 |
| ③ | **실행 (Execute)** | ALU에서 실제 연산 수행, 결과를 누산기(ACC)에 저장 |
| ④ | **저장 (Store)** | ACC → MBR → 메모리에 결과 저장 |

> [!note] 메모리 계층 구조 (빠름 → 느림)
> `레지스터 > 캐시(L1→L2→L3) > RAM > HDD/SSD`
> CPU는 데이터를 L1부터 순서대로 찾고, 없으면 아래 단계로 내려감

### ✅ 4.1 지역 변수, 범위, 지속시간 (C++)

**3가지 핵심 개념:**

| 개념 | 설명 |
|------|------|
| **범위 (Scope)** | 변수를 "볼 수 있는" 영역 → 선언 전·블록 밖 접근 불가 |
| **지속 기간 (Duration)** | 메모리가 살아있는 기간 → 블록 `{}` 끝나면 자동 반납 |
| **이름 숨기기 (Name Hiding)** | 안쪽 블록에 같은 이름 선언 시 바깥 변수가 가려짐 |

```cpp
int apple = 5;         // 바깥 블록
{
    int apple = 1;     // 안쪽 블록 → 바깥 apple을 숨김!
    cout << apple;     // 1 출력
}
cout << apple;         // 5 출력 (바깥 apple은 그대로)
```

> [!tip] 현대 C++ 원칙: 변수의 범위는 최대한 좁게!
> 사용하는 곳 근처에서만 선언 → 버그 ↓, 가독성 ↑ → OOP의 기초 철학

**네임스페이스 (Namespace):**

```cpp
namespace work1 { void doSomething() { a += 3; } }
namespace work2 { void doSomething() { a += 5; } }

work1::doSomething();  // work1의 것
work2::doSomething();  // work2의 것
// :: = 범위 결정 연산자 (Scope Resolution Operator)
```

> [!note] C++17 — 중첩 네임스페이스 간소화
> `namespace work1::work11::work111 { }` (depth는 1~2개가 적당)

## 2️⃣ Key Insights

- **CPU는 결국 수십억 개의 스위치** — 트랜지스터 ON/OFF가 모든 연산의 근원
- **버퍼 레지스터는 단순 중간 저장소가 아니다** — 속도·전압·폭이 다른 두 세계를 이어주는 인터페이스
- **인출→해석→실행→저장** 4단계는 CPU가 1초에 수십억 번 반복하는 핵심 사이클
- **C++ 스코프는 단순 규칙이 아니라 설계 철학** — 최소 범위 원칙이 OOP로 이어짐
