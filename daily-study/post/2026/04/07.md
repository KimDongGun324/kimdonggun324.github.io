## 1️⃣ Today I Learned

### ✅ cos/sin — 단위원과 벡터 방향

-   **cos = x좌표, sin = y좌표** (단위원 기준, 빗변=1)
-   직각삼각형 비율 패턴에 이름을 붙인 것 — 자연에서 발견된 것

$$\\cos^2\\theta + \\sin^2\\theta = 1 \\quad \\leftarrow \\text{피타고라스 정리 그 자체}$$

| $\\theta$ | $\\cos\\theta$ | $\\sin\\theta$ |
| --- | --- | --- |
| 0° | 1.0 | 0.0 |
| 45° | $\\approx$ 0.707 | $\\approx$ 0.707 |
| 90° | 0.0 | 1.0 |

> \[!tip\] 외우지 말고 유도
> 
> -   30/60/90: 정삼각형(변=2)을 반으로 자르면 빗변=2, 높이=1, 밑변=√3
> -   45/45/90: 정사각형(변=1) 대각선 → 빗변=√2

---

### ✅ 라디안(Radian) — HLSL 필수 지식

> \[!warning\] HLSL/GLSL은 **라디안** 사용. 도(°)를 넣으면 완전히 다른 값!

$$\\text{라디안} = \\text{도} \\times \\frac{\\pi}{180}$$

```
float angle_rad = radians(45.0);   // 45° → 0.785
float result = cos(angle_rad);     // 0.707
```

| 도(°) | 라디안 |
| --- | --- |
| 90° | π/2 ≈ 1.571 |
| 180° | π ≈ 3.14159 |
| 360° | 2π ≈ 6.283 |

### ✅ 내적(Dot Product) = cosθ

$$\\vec{a} \\cdot \\vec{b} = |\\vec{a}||\\vec{b}|\\cos\\theta \\quad \\xrightarrow{\\text{단위벡터}} \\quad \\cos\\theta$$

> **내적 = 두 방향이 얼마나 일치하는가** (−1 ~ 1)

-   같은 방향: `dot = 1.0` / 수직: `dot = 0.0` / 반대: `dot = -1.0`
-   sin, tan을 안 쓰는 이유: sin은 값이 반대(수직 입사=0), tan은 90°에서 발산

**⚠️ 항상 normalize() 먼저!**

```
// 크기가 1이 아닌 벡터로 내적하면 cosθ가 아님
float3 L = normalize(lightDir);
float3 N = normalize(normal);
float NdotL = dot(N, L);   // = cosθ ✅
```

### ✅ Lambert's Cosine Law & Diffuse 조명

$$I = I\_0 \\cdot \\cos\\theta$$

비스듬할수록 같은 에너지가 더 넓은 면적에 퍼짐 → 단위면적당 강도 감소

```
// Lambertian Diffuse 구현
float diffuse = max(0.0, dot(normal, lightDir));
// max(0, ...) : 뒷면(cosθ < 0) 차단
```

| 각도 | cosθ | 상황 |
| --- | --- | --- |
| 0° | 1.0 | 수직 입사 → 최대 |
| 60° | 0.5 | 비스듬히 입사 → 절반 |
| 90°+ | ≤ 0 | 뒷면 → `max`로 차단 |

### ✅ 조명 3요소 — Phong Lighting Model

$$\\text{최종 색상} = \\underbrace{\\text{Ambient}}_{\\text{간접광}} + \\underbrace{\\text{Diffuse}}_{\\text{난반사}} + \\underbrace{\\text{Specular}}\_{\\text{정반사}}$$

```
// Ambient: 방향 무관, 기저 밝기
float3 ambient  = ambientStrength * lightColor;

// Diffuse: Lambert's Law
float NdotL     = max(dot(normal, lightDir), 0.0);
float3 diffuse  = NdotL * lightColor;

// Specular: Blinn-Phong (Half Vector)
float3 halfDir  = normalize(lightDir + viewDir);
float spec      = pow(max(dot(normal, halfDir), 0.0), shininess);
float3 specular = specularStrength * spec * lightColor;

float3 result   = (ambient + diffuse + specular) * objectColor;
```

> \[!note\] Blinn-Phong vs Phong
> 
> -   **Phong**: reflect(-L, N) vs ViewDir — 90° 넘으면 하이라이트 갑자기 사라짐
> -   **Blinn-Phong**: normalize(L+V) = HalfVector vs Normal — 더 안정적, 실무 표준

### ✅ cos/sin의 주기성 — 셰이더 애니메이션

```
// cos, sin은 2π(360°)마다 반복 → 자연스러운 주기 운동
float2 circlePos = float2(cos(time), sin(time));  // 원형 궤도

// 0~1 범위로 변환 (UV 애니메이션 등)
float wave = sin(time) * 0.5 + 0.5;   // -1~1 → 0~1
```

## 2️⃣ Key Insights

-   **cos/sin은 발명이 아니라 발견** — 직각삼각형 비율의 패턴이 먼저 있었고, 이름이 붙었다
-   **내적 = cosθ** 가 성립하려면 반드시 단위벡터여야 함 → `normalize()` 빠뜨리면 밝기가 틀림
-   **Lambert's Law는 물리 법칙** (1760년) — "비스듬할수록 넓은 면적에 퍼진다"는 직관이 cosθ로 표현됨
-   **조명 = Ambient + Diffuse + Specular** — 각각 독립적으로 계산 후 합산
-   **HLSL은 라디안** — 이걸 모르면 cos(45) = cos(45 radian) = 0.526 (45°의 0.707이 아님!)
