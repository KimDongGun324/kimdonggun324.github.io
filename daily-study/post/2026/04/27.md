## 1️⃣ Today I Learned

### 포인터의 본질 — 주소를 담는 변수

```cpp
int x = 5;
// 내부적으로 하는 일:
// 1. OS로부터 4바이트 스택 메모리 공간을 빌려옴
// 2. 그 공간에 고유한 주소(address)가 부여됨 — 예: 0x009EF764
// 3. 그 주소에 값 5를 저장
```

```
RAM: [addr 0x0000]...[addr 0x009EF764][int x: 5]...[addr 0xFFFFFFFF]

CPU가 x의 값이 필요할 때:
  메모리 전체를 처음부터 뒤지는 것이 아니라
  → '주소 0x009EF764로 직접 이동해' 4바이트를 읽어옴
  → 이것이 주소 기반 직접 접근의 핵심
```

포인터는 이 주소를 **변수로서** 직접 다루는 도구다. 포인터도 결국 주소를 저장하는 변수에 불과하다.

### Address-of 와 De-reference — 두 핵심 연산자

```cpp
int x = 5;

// & : 변수의 메모리 주소를 반환 (address-of operator)
cout << x      << endl;   // 5         — 값(value)
cout << &x     << endl;   // 009EF764  — 주소(16진수)
cout << (int)&x << endl;  // 10416996  — 주소(10진수)

// * : 주소에 있는 값에 접근 (dereference operator)
int* ptr_x = &x;
cout << ptr_x  << endl;   // 009EF764  — 포인터가 담은 주소
cout << *ptr_x << endl;   // 5         — 그 주소에서 꺼낸 값

// 포인터로 값 변경 — dereference 쓰기
*ptr_x = 99;
cout << x << endl;        // 99  — x의 값이 바뀌었다!
```

> [!tip] De-reference의 의미
> 포인터가 "저쪽 주소에 가면 이 데이터가 있어요"라고 **간접적으로 가리키기만** 하는 것에 대해,
> De-reference는 **"그럼 거기에 진짜 뭐가 있는지 내가 들여다볼게"** 라며 직접 접근하겠다는 선언이다.

**연산자 우선순위:** `()` > `*` (단항) > `&` (단항)

```cpp
cout << *(&x) << endl;   // 5 — 괄호 먼저, 그 다음 역참조
cout << *&x   << endl;   // 5 — 동일 (& 먼저, * 나중) — *& 는 서로 상쇄
```

### sizeof 포인터 — 타입 무관 고정 크기

```cpp
int*       ptr_x;   // sizeof: 4 (32비트) 또는 8 (64비트)
double*    ptr_d;   // sizeof: 동일
char*      ptr_c;   // sizeof: 동일

struct Something { int a, b, c, d; };  // 16바이트
Something* ptr_s;   // sizeof: 4 (구조체 크기와 무관!)

cout << sizeof(Something) << endl;  // 16
cout << sizeof(ptr_s)     << endl;  // 4
```

| 환경 | 포인터 크기 | 이유 |
|------|------------|------|
| 32비트 | 4바이트 | 2³² 주소 공간 표현 |
| 64비트 | 8바이트 | 2⁶⁴ 주소 공간 표현 |

> [!important] 포인터의 크기는 가리키는 데이터 타입과 무관하다
> 포인터는 **주소**를 저장하고, 주소 자체의 크기는 CPU 아키텍처에 의해 결정된다.
> 집 주소의 길이가 집 크기와 무관하듯, 포인터 크기는 가리키는 데이터 크기와 무관하다.

---

### 포인터 타입이 필요한 두 가지 이유

```cpp
int*    ptr_x = &x;   // int*    : "4바이트를 읽어라"
double* ptr_d = &d;   // double* : "8바이트를 읽어라"
```

> [!important] 왜 `int*`과 `double*`을 구분해야 하는가
>
> **① De-reference 시 읽을 바이트 수 결정**
> 메모리 주소는 그냥 숫자일 뿐이다. `int*`이면 그 주소에서 4바이트를 읽고 int로 해석,
> `double*`이면 8바이트를 읽고 double로 해석한다. 타입이 없으면 컴파일러가 알 수 없다.
>
> **② 포인터 산술 시 이동 단위 결정**
> ```cpp
> int arr[3] = {1, 2, 3};
> int* p = arr;
> ++p;   // int* : 4바이트 전진 — 다음 int로 이동
> ```

### 미초기화 포인터 — 가장 흔한 런타임 오류

```cpp
int* ptr_x;            // 선언만 하고 초기화 안 함
                       // ptr_x 안에는 쓰레기 값(임의의 주소) 존재

cout << *ptr_x << endl;   // 런타임 오류!
```

```
실제 오류 메시지:
Run-Time Check Failure #3
The variable 'ptr_x' is being used without being initialized.
```

```
ptr_x 안의 쓰레기 값 (예: 0xCCCCCCCC)
    ↓
*ptr_x → 0xCCCCCCCC 주소로 이동 시도
    ↓
OS: "그 주소는 이 프로그램에게 허용된 메모리가 아님!"
    ↓
Access Violation / Segmentation Fault → 강제 종료
```

> [!danger] 교수님: "이게 제일 많이 발생하는 오류입니다"
> 쓰레기 주소를 역참조하면 **감지되지 않고 조용히 메모리를 오염시킬 수 있다**는 점이 더 큰 문제다.
> 반대로 내 프로그램이 사용하지 않는 메모리 주소를 억지로 접근하는 것이 **해킹의 원리**이기도 하다.

### nullptr — C++11 타입 안전 널 포인터

```cpp
double* ptr1 = NULL;       // C 스타일 — 매크로 (#define NULL 0)
double* ptr2{ 0 };         // 정수 리터럴 0의 암묵적 변환
double* ptr3{ nullptr };   // C++11 이후 권장 — 타입 안전
```

`nullptr`은 C++11에서 도입된 **키워드**다. `std::nullptr_t` 타입을 가지며, 어떤 포인터 타입으로도 암묵적 변환되지만 **정수로는 변환되지 않는다** → 타입 안전성 보장.

---

### nullptr vs NULL vs 0 — 타입 시스템의 차이

| 방법 | 도입 | 타입 | 정수 변환 | 오버로딩 안전 |
|------|------|------|-----------|--------------|
| `0` | C 시대 | `int` | 가능 | ⚠️ 위험 |
| `NULL` | C 매크로 | 구현마다 다름 | 가능 | ⚠️ 위험 |
| `nullptr` | C++11 | `std::nullptr_t` | **불가** | ✅ 안전 |

**오버로딩 함정:**

```cpp
void foo(int x)   { cout << "int 버전" << endl; }
void foo(int* p)  { cout << "포인터 버전" << endl; }

foo(NULL);     // 의도는 포인터이지만, int 버전이 호출될 수 있다!
foo(nullptr);  // 항상 포인터 버전 호출 — 명확
```

### null 체크 패턴 — 방어적 프로그래밍

```cpp
void doSomething(double* ptr)
{
    if (ptr != nullptr)
    {
        std::cout << *ptr << std::endl;  // 유효한 주소만 역참조
    }
    else
    {
        std::cout << "Null ptr, do nothing" << std::endl;
    }
}
```

이를 **가드 조건문(guard clause)** 또는 **방어적 프로그래밍(defensive programming)** 이라 부른다.

> [!note] Early Return 패턴 — 더 간결한 실무 스타일
> ```cpp
> void doSomething(double* ptr)
> {
>     if (ptr == nullptr) return;   // null이면 즉시 반환
>     std::cout << *ptr << std::endl;
> }
> ```
> if 블록 깊이를 줄여 가독성이 높아진다. 실무 코드베이스에서 더 자주 보이는 스타일이다.

### std::nullptr_t — nullptr 전용 타입

```cpp
#include <cstddef>   // std::nullptr_t 여기 정의됨

std::nullptr_t nptr; // nullptr 타입의 변수 — nullptr만 담을 수 있다
```

`std::nullptr_t`는 nullptr 전용 함수 오버로드가 필요할 때 사용한다:

```cpp
void process(double* ptr)
{
    std::cout << "double pointer: " << *ptr << "\n";
}

void process(std::nullptr_t)           // nullptr 전용 오버로드
{
    std::cout << "nullptr received, skipping\n";
}

process(nullptr);  // nullptr_t 오버로드 호출 — 명확하게 분리됨
```

> [!info] 교수님이 "한 번도 안 써봤다"고 하신 이유
> 일반적인 애플리케이션 코드에서는 거의 사용하지 않는다.
> 주로 라이브러리 설계·템플릿 메타프로그래밍·SDK 인터페이스 설계 시 드물게 등장한다.
