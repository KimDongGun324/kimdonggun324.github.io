### ✅ Python 자료형 완성 — 불 · 집합 · 변수 02-6 ~ 02-8

#### 불 자료형(bool) — Truthy와 Falsy

파이썬의 if 문은 조건 자리에 어떤 객체든 올 수 있다. 파이썬이 자동으로 불리언으로 변환해 참/거짓을 판단한다.

**Falsy 값 — if 조건에서 False로 평가되는 것:**

| 값 | 타입 |
|---|---|
| `False` | bool |
| `None` | NoneType |
| `0`, `0.0` | int, float |
| `""` | 빈 문자열 |
| `[]`, `()`, `{}`, `set()` | 빈 컬렉션 |

```python
# Pythonic한 빈 리스트 검사
if my_list:          # len(my_list) > 0 대신
    process(my_list)

# None 검사는 반드시 is 사용 — PEP 8 권장
if result is None:   # if not result: 는 0, 빈 리스트도 걸러내므로 위험
    handle_none()
```

#### 집합 자료형(set) — 중복 제거 + 집합 연산

```python
# 집합 생성 — 중복 자동 제거, 순서 없음
s1 = {1, 2, 3}
s2 = set([1, 2, 3, 3, 4])  # {1, 2, 3, 4} — 중복 제거

# 핵심 집합 연산
a = {1, 2, 3, 4, 5}
b = {3, 4, 5, 6, 7}

print(a & b)   # 교집합 → {3, 4, 5}
print(a | b)   # 합집합 → {1, 2, 3, 4, 5, 6, 7}
print(a - b)   # 차집합 → {1, 2}
print(a ^ b)   # 대칭차 → {1, 2, 6, 7}

# in 연산 — O(1) (리스트 O(n)과 대조)
valid_formats = {'PNG', 'JPG', 'EXR', 'HDR', 'TGA'}
if 'EXR' in valid_formats:   # 해시 테이블 O(1)
    print("유효한 포맷")
```

> [!tip] set의 `in` 연산은 O(1)
> 자주 멤버십 검사가 필요하면 리스트를 set으로 변환하는 것이 성능상 유리하다. 유효 포맷 목록, 처리된 파일명 중복 체크 등에서 실무 활용도가 높다.

### ✅ Python 조건문 if문 03-1

#### if / elif / else 기본 구조와 실행 흐름

```python
# elif: 앞 조건이 참이면 뒤는 평가조차 안 함
score = 85

if score >= 90:       # 거짓 → 건너뜀
    grade = "A"
elif score >= 80:     # 참 → 실행, 이후 모두 건너뜀
    grade = "B"
elif score >= 70:     # 평가 안 됨
    grade = "C"
else:                 # 실행 안 됨
    grade = "F"

print(grade)  # "B"
```

**들여쓰기 규칙:**
- 공백 4칸 권장 (PEP 8)
- 탭과 스페이스 혼용 금지 → `TabError`
- 블록은 반드시 같은 깊이 유지

#### match-case — 구조적 패턴 매칭 (Python 3.10+)

```python
# 단순 값 비교 — elif 체인보다 간결
grade = "B"
match grade:
    case "A":
        print("탁월한 성적입니다.")
    case "B" | "C":      # OR 패턴 — 여러 값 묶기
        print("우수/보통입니다.")
    case _:              # 와일드카드 — else와 동일
        print("노력이 필요합니다.")

# 구조적 패턴 매칭 — 튜플 분해 (강력한 기능)
def classify_vector(v):
    match v:
        case (0, 0, 0):
            return "영벡터"
        case (x, 0, 0):
            return f"X축 ({x})"
        case (x, y, z):
            return f"일반 벡터 ({x}, {y}, {z})"
```

#### *args / **kwargs — 가변 인수

```python
# *args: 위치 인수 → 튜플로 받음
def add_many(*args):
    result = 0
    for i in args:
        result += i
    return result

print(add_many(1, 2, 3, 4, 5))   # 15

# **kwargs: 키워드 인수 → 딕셔너리로 받음
def print_kwargs(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_kwargs(name="홍길동", age=25, city="서울")

# 혼합 사용 순서 — 반드시 이 순서를 지켜야 함
def mixed(a, b, *args, **kwargs):
    print(a, b, args, kwargs)

mixed(1, 2, 3, 4, 5, x=10, y=20)
# a=1, b=2, args=(3,4,5), kwargs={'x':10,'y':20}
```

#### LEGB 스코프 규칙 — 변수 탐색 순서

```python
x = "전역"   # Global

def outer():
    x = "인클로징"  # Enclosing

    def inner():
        x = "지역"  # Local
        print(x)    # "지역" — L(Local) 우선
    inner()
    print(x)        # "인클로징" — E(Enclosing) 다음

outer()
print(x)            # "전역" — G(Global)

# global 키워드 — 전역 변수 수정 (남용 금지)
counter = 0
def increment():
    global counter
    counter += 1

# nonlocal 키워드 — 인클로징 스코프 변수 수정
def make_counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment
```

### ✅ Python 파일 입출력 04-3

#### open() — 파일 열기 모드

| 모드 | 설명 | 주의 |
|------|------|------|
| `'r'` | 읽기 (기본값) | 파일 없으면 `FileNotFoundError` |
| `'w'` | 쓰기 (덮어쓰기) | 기존 내용 전부 삭제 |
| `'a'` | 추가 쓰기 | 기존 내용 유지, 끝에 추가 |
| `'rb'` / `'wb'` | 바이너리 읽기/쓰기 | 이미지, 바이너리 파일 처리 시 |

```python
# 파일 쓰기
f = open("output.txt", 'w', encoding='utf-8')
for i in range(1, 6):
    f.write(f"{i}번째 줄\n")
f.close()

# 파일 읽기 — 3가지 방법
f = open("output.txt", 'r', encoding='utf-8')
line   = f.readline()       # 한 줄 읽기
lines  = f.readlines()      # 전체 → 리스트 (줄 단위)
content = f.read()           # 전체 → 문자열
f.close()
```

#### with 문 — RAII 패턴 (권장)

```python
# with 문: 블록 종료 시 자동으로 close() 호출
# 예외 발생해도 반드시 닫힘 → C++의 RAII와 동일한 원리
with open("output.txt", 'r', encoding='utf-8') as f:

        print(line.strip())           # strip(): 양쪽 공백/\n 제거

# 쓰기 + 읽기
with open("data.txt", 'w', encoding='utf-8') as f:
    f.write("line 1\nline 2\nline 3\n")

with open("data.txt", 'r', encoding='utf-8') as f:
    data = [line.strip() for line in f]   # 리스트 컴프리헨션으로 한 번에
print(data)  # ['line 1', 'line 2', 'line 3']
```

### ✅ 그래픽스 엔지니어 셰이더 디자이너 관점

#### Python 함수 → GLSL/HLSL 함수 설계 패턴

파이썬의 함수 설계 원칙(단일 책임, 명확한 입출력, 재사용)은 셰이더 함수 설계에도 그대로 적용된다. GLSL/HLSL에서 PBR 계산을 `DistributionGGX()`, `GeometrySmith()`, `FresnelSchlick()` 같은 순수 함수로 분리하는 것은, 파이썬에서 하나의 함수가 하나의 역할만 하는 원칙의 셰이더 버전이다. 파이썬으로 셰이더 로직을 CPU에서 먼저 검증하고(`*args`로 여러 파라미터를 유연하게 받는 함수), 검증된 수식을 GLSL/HLSL로 포팅하는 워크플로우는 그래픽스 엔지니어에게 매우 효율적인 개발 방식이다.

#### if 분기 발산(SIMD Divergence) — GPU에서 if 문의 비용

GPU는 수천 개의 스레드가 완전히 같은 명령을 동시에 실행(SIMD)하는 방식으로 동작한다. 한 워프(warp, 32개 스레드) 안에서 어떤 스레드는 if 블록, 다른 스레드는 else 블록을 실행해야 하면, GPU는 두 경로를 **순차적으로 모두 실행하고 불필요한 결과를 버린다** — 이것이 분기 발산(Branch Divergence)이다. 셰이더 최적화의 핵심 목표 중 하나가 if 문을 `step()` / `smoothstep()` / `mix()` 같은 수학 함수로 대체하는 것이며, 파이썬에서 조건부 표현식(`값 if 조건 else 값`)을 수학 연산으로 바꾸는 연습이 이 감각을 기른다.

```python
# Python — 분기 없이 수학으로 조건 표현 (GPU 셰이더와 동일 원리)
def step(edge, x):
    """GLSL step(): x >= edge이면 1.0, 아니면 0.0"""
    return 1.0 if x >= edge else 0.0

def smoothstep(e0, e1, x):
    """GLSL smoothstep(): Hermite 보간"""
    t = max(0.0, min(1.0, (x - e0) / (e1 - e0)))
    return t * t * (3.0 - 2.0 * t)

def mix(a, b, t):
    """GLSL mix(): 선형 보간"""
    return a + (b - a) * t
```

#### for 루프와 배치 텍스처 처리 — 에셋 파이프라인

for 루프와 리스트 컴프리헨션은 그래픽스 엔지니어의 에셋 파이프라인에서 핵심 도구다. 수백 개의 텍스처를 순회하며 포맷 변환, 밉맵 생성, 압축 적용, 메타데이터 추출을 자동화하는 파이프라인이 `for file in texture_files:` 루프 하나에서 시작된다. 특히 `zip()`으로 텍스처 이름과 설정값을 병렬 순회하거나, 리스트 컴프리헨션으로 EXR 파일만 필터링하는 패턴은 실무에서 매일 사용되는 코드다.

```python
import os

# 실무 에셋 파이프라인 — for + 리스트 컴프리헨션
texture_dir = "Assets/Textures"
all_files   = os.listdir(texture_dir)

# EXR 파일만 필터링 — set 멤버십 O(1)
raw_formats = {'exr', 'hdr', 'tiff'}
exr_files   = [f for f in all_files if f.split('.')[-1].lower() in raw_formats]

# 알베도/노말/러프니스 분류 — enumerate로 번호 부여
for i, fname in enumerate(exr_files, 1):
    if 'albedo' in fname.lower():
        print(f"[{i}] 알베도 맵: {fname}")
    elif 'normal' in fname.lower():
        print(f"[{i}] 노말 맵: {fname}")

# 머티리얼 파라미터 배치 처리 — zip 활용
mat_names  = ['rock', 'wood', 'metal']
roughness  = [0.8, 0.6, 0.1]
metallic   = [0.0, 0.0, 1.0]
for name, r, m in zip(mat_names, roughness, metallic):
    print(f"{name}: roughness={r}, metallic={m}")
```

#### 파일 I/O — 셰이더 소스 로딩과 with 문의 RAII

셰이더 엔진에서 GLSL/HLSL 소스 파일을 로드하는 작업은 Python의 파일 I/O와 동일한 원리다. `with open()` 의 자동 close 보장은 C++의 RAII(Resource Acquisition Is Initialization) 패턴과 정확히 같은 개념이며, 파일 핸들 누수 없이 셰이더 소스를 안전하게 읽을 수 있다. 실무에서는 셰이더 파라미터, 머티리얼 설정, 렌더 설정을 JSON 파일로 저장하고 `with open()` + `json.load()`로 읽는 패턴이 표준이다.

```python
import json

# 셰이더 소스 파일 로딩 — with로 안전하게
with open("shaders/pbr.glsl", 'r', encoding='utf-8') as f:
    shader_source = f.read()

# 머티리얼 설정 JSON 읽기 — 파일 I/O + 딕셔너리 결합
with open("materials/rock.json", 'r', encoding='utf-8') as f:
    material = json.load(f)

roughness = material.get('roughness', 0.5)
metallic  = material.get('metallic',  0.0)
albedo    = material.get('albedo',    [1.0, 1.0, 1.0])
```

#### 변수의 참조 개념 — GPU 메모리 포인터와 유사성

파이썬에서 변수가 객체를 가리키는 이름표(reference)라는 개념은, GPU 프로그래밍에서 텍스처 핸들(handle)이나 버퍼 포인터가 실제 VRAM 데이터를 가리키는 방식과 개념적으로 동일하다. OpenGL의 `GLuint textureID`는 실제 텍스처 데이터가 아닌 GPU 리소스를 가리키는 ID(reference)다. 파이썬 리스트 복사에서 얕은 복사(shallow copy)와 깊은 복사(deep copy)의 차이를 이해하면, GPU에서 버퍼를 공유할 때와 독립 복사본을 만들 때의 차이(aliasing vs. independent resource)를 직관적으로 이해할 수 있다.

## 2️⃣ Key Insights

- **Truthy/Falsy는 if 조건의 핵심 — None과 빈 컬렉션을 구분하라** — `if not result:`는 None, 0, 빈 리스트를 모두 같이 처리한다. 그래픽스 파이프라인에서 텍스처가 로드되지 않았는지(None) vs 유효한 데이터가 비어있는지를 구분해야 할 때 `is None` 검사가 필수다

- **match-case는 단순 값 비교를 넘어 구조 분해까지 가능하다** — Python 3.10+의 구조적 패턴 매칭은 렌더 커맨드 타입에 따른 분기, API 응답 구조 파싱 등에서 if-elif 체인보다 훨씬 명확하다

- **리스트 컴프리헨션은 에셋 파이프라인의 표준 도구다** — `[f for f in files if f.endswith('.exr')]` 한 줄이 수백 개 파일 필터링을 완성한다. 가독성과 성능을 동시에 잡는다

- ***args / **kwargs가 유연한 API 설계의 핵심** — 셰이더 파라미터, 렌더 설정처럼 파라미터 수가 유동적인 경우에 **kwargs로 키워드 인수를 딕셔너리로 받으면 확장성이 뛰어나다

- **LEGB 스코프는 버그의 근원이기도, 강력한 도구이기도 하다** — 클로저(closure)로 상태를 캡슐화하는 패턴(nonlocal)과 전역 변수 남용의 위험성을 함께 이해해야 한다. 셰이더 에디터 도구에서 콜백 함수가 외부 상태를 캡처하는 패턴이 바로 클로저다

- **with 문은 Python의 RAII — 예외가 나도 반드시 닫힌다** — 파일, 네트워크 연결, GPU 컨텍스트 같은 자원은 항상 with 문이나 try-finally로 해제를 보장해야 한다. C++의 unique_ptr과 동일한 철학이다

- **GPU에서 if 문은 비용이 크다 — 수학으로 대체하라** — 분기 발산(Branch Divergence)으로 인해 GPU 셰이더에서 if 문은 두 경로를 모두 실행하고 버린다. `step()` / `smoothstep()` / `mix()` 함수가 if 없이 조건을 수학으로 표현하는 핵심이며, 파이썬에서 이 로직을 먼저 검증하는 연습이 중요하다

- **집합(set)의 O(1) 멤버십 검사를 활용하라** — 유효 포맷 목록, 처리된 파일명 중복 체크, 렌더 패스 활성화 여부 등 "있는지 없는지"를 자주 확인해야 하는 데이터는 list 대신 set으로 관리해야 한다
