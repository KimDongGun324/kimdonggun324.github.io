  <h1 style="margin-top: 20px;">🛠️ 깃허브 블로그 구축과 CSS 트러블 슈팅</h1>
  <p style="color: #666; font-size: 0.9em;">2025.11.27 | Category: Web / Git</p>

  <hr style="margin: 30px 0; border: 0; border-top: 1px solid #e5e5e5;">

  <h3>1. Problem (문제 상황)</h3>
  <p>
    포트폴리오용 깃허브 블로그를 구축하던 중, <code>assets/css/style.scss</code> 파일의 경로를 제대로 인식하지 못해
    디자인이 적용되지 않고 HTML 뼈대만 나오는 현상이 발생했다.
  </p>

  <h3>2. Try (시도한 방법)</h3>
  <ul>
    <li>파일 경로를 <code>assets/style.scss</code>에서 <code>assets/css/style.scss</code>로 변경해 보았다.</li>
    <li>Jekyll 테마 설정을 확인해 보았다.</li>
  </ul>

  <h3>3. Solution (해결)</h3>
  <p>
    결국 외부 CSS 파일을 불러오는 방식 대신, <strong>HTML 레이아웃 파일(<code>_layouts/default.html</code>) 내부에
    <code>&lt;style&gt;</code> 태그를 사용하여 CSS를 직접 주입하는 방식</strong>으로 변경하여 해결했다.
    이 방식은 파일 경로 문제에서 자유롭고, 확실하게 스타일을 적용할 수 있다는 장점이 있었다.
  </p>

  <h3>4. Insight (배운 점)</h3>
  <p>
    개발 환경(Jekyll)의 특성을 이해하지 못하면 단순한 경로 문제로도 많은 시간을 쏟을 수 있다는 것을 깨달았다.
    앞으로는 문제 발생 시 <strong>"가장 확실하고 단순한 방법(Inline Style)"</strong>을 먼저 고려하여 효율을 높여야겠다.
  </p>
